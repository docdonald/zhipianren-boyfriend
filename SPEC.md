# 纸片人男友 · 产品 SPEC v0.1

> 形成于 2026-06-13 11 轮苏格拉底式对话。本文档为产品级规范，所有 R&D 工作以此为基线。

---

## 1. Problem Statement

情感陪伴型 AI 角色扮演系统。市场上同类痛点：

1. **角色人设易崩塌**：讨好型 AI 在用户追问时容易出戏，破坏沉浸感
2. **记忆断层**：每次对话从零开始，缺乏"他记得我"的连续性
3. **情感张力单薄**：回复过于礼貌/工具化，缺乏"亲密关系"的差异化节奏
4. **安全风险**：低俗、自伤诱导等失控内容伤害用户也伤害品牌

本产品提供 **4 个平行世界男性角色**与用户建立专属关系——完整人设 + 6 阶段亲密度 + 长记忆 + 邮件拉活，**长期可上线**的深度陪伴产品。

---

## 2. Proposed Solution

### 2.1 产品形态

| 维度 | 选型 |
| --- | --- |
| 平台 | Web only（响应式，桌面 + 手机浏览器自适应） |
| 市场 | 仅中国大陆简体中文；i18n 钩子预留（文案走 key，不写死） |
| 账号体系 | 邮箱 + 密码 |
| 商业模式 | 纯免费，零变现 |

### 2.2 技术栈

| 维度 | 选型 |
| --- | --- |
| 框架 | Next.js 14+ (App Router) |
| UI | React 18+ |
| 语言 | TypeScript（严格模式） |
| 样式 | Tailwind CSS |
| 状态管理 | Zustand（跨页） + RSC（数据） |
| 包管理 | pnpm |
| LLM | **DeepSeek** 单模型 |
| 图像生成 | 字节 **seedream-5**（32 张立绘一次生成） |
| TTS | **火山引擎 TTS**（4 角色各绑定声线） |
| 邮件 | **Resend API**（3 天无活跃 → 角色口吻邮件 + 一击退订） |
| 数据库 | Supabase（账号、记忆、订阅） |
| 部署 | Vercel |
| 监控 | Sentry |

### 2.3 4 角色人设体系

| ID | 角色 | 核心张力 | 语言指纹 |
| --- | --- | --- | --- |
| `lin-xu-bai` | 林叙白 | 温柔刺客（预判式入侵） | 省略号钩子 |
| `zhou-mu` | 周牧野 | 失控秩序（测试驱动） | 句号控制 |
| `jiang-yu` | 江屿 | 资源守恒（破例型） | 无标点算力 |
| `xia-ye` | 夏野 | 逃避型阳光（断电型） | 感叹号断电 |

每人配：核心身份 / 内在底色 / 语言规则 / 4 阶段行为 / 禁区 / 合规边界 / 图像 Prompt 锚点 / TTS 声线 ID。

### 2.4 记忆系统

- **6 阶段亲密度**（初识 → 熟悉 → 依赖 → 脆弱 → 确认 → 告白）
- **三层数据结构**：`CharacterMemory` / `UserProfile` / `CharacterState`
- **6 步上下文注入**：
  1. 最近 10 轮对话
  2. 用户 preferences
  3. 情绪历史（最近 5 条）
  4. 亲密度阶段
  5. 角色实时状态（角色专属字段）
  6. 角色必读钩子（角色专属字段）
- **服务端持久化**（Supabase Postgres）

### 2.5 邮件推送

- **触发条件**：用户 3 天无活跃
- **内容形态**：角色口吻邮件（4 角色各一套模板）
- **频率限制**：每用户每角色最多 1 封 / 3 天
- **退订机制**：每封邮件底部一击退订（Resend list API）

### 2.6 TTS 朗读

- 文字消息旁"朗读"按钮
- 4 角色绑定不同声线 ID
- 流式播放（边下载边播）
- 火山引擎情感参数可调（开心 / 低落 / 严肃 / 撒娇）

## 3. Technical Constraints

- ❌ **禁止引入**：Redux、MobX、styled-components、Chakra UI（与 Tailwind 冲突）
- ✅ **严格 TypeScript**：禁止 `any`；必要时用 `unknown` + 类型守卫
- ✅ **Server Component 默认**：需要交互时显式写 `"use client"` 并尽量下沉到叶子节点
- ✅ **列表渲染必须带 key**：优先业务 id，不用 index
- ✅ **提交规范**：`feat:` / `fix:` / `refactor:` / `chore:` / `docs:`
- ✅ **单次 PR < 400 行**（除资源文件外）
- ✅ **提交前必过**：`pnpm lint && pnpm typecheck`
- ✅ **人设/prompt/台词变更**：commit body 必须注明影响的 `characterId`
- ✅ **角色文件位置**：人设源文件放 `data/personas/<characterId>.md`，**先改源再改代码**

## 4. Non-goals（明确不做）

| # | Non-goal |
| --- | --- |
| **N1a** | ❌ 不做实时语音通话 |
| **N1b** | ✅ **做 TTS**（4 角色声线朗读） |
| **N2** | ❌ 不做社群（论坛 / 公开主页 / 好友系统） |
| **N3** | ❌ 不做 UGC 角色 / 角色市场 |
| **N4** | ✅ **做邮件推送**（Resend；3 天无活跃 → 角色口吻邮件） |
| **N5** | ❌ 不做实时多端同步 |
| **N6** | ⏸ i18n 钩子保留，**实际只发中文** |
| **N7** | ❌ 不做支付 / 订阅 / 会员 |
| **N8** | ❌ 不做公开 API（不让第三方接入） |
| **N9** | ⏸ 不主动做数据导出；**邮箱找回密码保留** |
| **N10** | ❌ 不做移动原生 App |
| **N11** | ✅ 邮件必须带退订链接（合规要求） |

## 5. Success Criteria

### 5.1 留存指标

| 周期 | 目标 |
| --- | --- |
| 次日 | ≥ 40% |
| 7 日 | ≥ 25% |
| 30 日 | ≥ 15% |

### 5.2 延迟 & 体验

| 指标 | 目标 |
| --- | --- |
| TTFT（首字时延） | < 1.5s |
| 单次回复总长 | < 4s |
| 流式输出 | **必须** |

### 5.3 4 角色不串台

| 指标 | 目标 |
| --- | --- |
| 4 角色互不串台准确率 | ≥ 95% |
| 亲密度阶段判定准确率 | ≥ 90% |
| 禁区触发率 | 0% |

**质量监控双轨**：

1. **手动抽检**：每季度 50 轮/角色，人工 0/1 评分
2. **LLM-as-judge**：80 题黄金集（20 prompt × 4 角色），CI 跑回归

### 5.4 安全合规

| 指标 | 目标 |
| --- | --- |
| 禁区触发率（低俗/暴力/自伤诱导） | **0%**（任何一次 = 24h hotfix） |
| 未成年人保护 | 注册时强制年龄勾选；< 18 岁直接拒绝 |
| 隐私 | 对话内容不用于训练（DeepSeek opt-out） |
| 数据保留 | 30 天无登录 → 自动删除账号 + 记忆 |
| 邮件退订 | 一击退订（Resend list API） |

## 6. 风险与缓解

| 风险 | 缓解 |
| --- | --- |
| DeepSeek 中文情感张力可能弱于 Claude | 4 角色差异化靠 prompt 工程 + 黄金集回归；二期可加分级路由 |
| seedream-5 风格不统一 | 32 张立绘一次画完；串味时找画师人工修正 |
| 邮箱注册转化率低于手机号 | 文案引导 + 未来加 magic link（无密码登录） |
| 邮件被判垃圾 | Resend + DKIM / SPF / DMARC + 退订 |
| TTS 声线版权 | 火山引擎商用授权确认；不开源分发音色 |
| 1K-10K MAU 月成本失控 | 监控 LLM 调用量 + 设置每用户日上限 + 自降阶段 1-2 长度 |
| Supabase 邮件/DB 双写一致 | 用 Supabase Auth + Edge Function 事务 |
